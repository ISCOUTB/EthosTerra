¿Qué motor de juego usamos dentro de Next.js?

Pixel Art isométrico · Panel lateral · Mundo con edificios y rutas

🎮
Phaser 3
phaser.io
ISOMÉTRICO ✓ TILES ✓ PIXEL ✓

A — Phaser 3

Motor 2D maduro con soporte nativo para tilemaps isométricos, sprites animados, pathfinding (plugin A*), y fácil integración en React via canvas. Ecosistema muy rico para juegos 2D.

Pros

  • Diseñado para esto exactamente
  • Tilemaps Tiled (.tmx) nativos
  • A* pathfinding disponible
  • Gran comunidad y docs

Contras

  • Integración con React requiere wrapper
  • Canvas separado del DOM
🎨
Canvas API
React + useRef + requestAnimationFrame
CONTROL TOTAL SIN DEPS

B — Canvas API puro

Renderizado isométrico hecho a mano con useRef y requestAnimationFrame. Control total, sin dependencias externas, fácil de integrar con el estado React. Mayor esfuerzo inicial.

Pros

  • Integración React perfecta
  • Sin dependencias pesadas
  • Control total del render

Contras

  • Hay que implementar todo desde cero
  • Pathfinding manual
  • Más tiempo de desarrollo
⚡
PixiJS 8
pixijs.com — WebGL acelerado
WEBGL +RÁPIDO SPRITES

C — PixiJS 8

Renderer WebGL de alto rendimiento ideal para muchos sprites simultáneos. Perfecto para 20-150 agentes animados. Más liviano que Phaser, más potente que Canvas puro.

Pros

  • Muy performante con muchos agentes
  • Sprites y animaciones pixel art
  • Integra bien con React

Contras

  • No tiene tilemaps isométricos nativos
  • Pathfinding hay que añadirlo

Mi recomendación: Phaser 3 — tiene todo lo que necesita este proyecto de caja.
